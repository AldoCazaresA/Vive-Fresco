Introducción

Proyecto: Vive Fresco

Equipo:

Aldo Cázares - Lider y desarrollador

Antecedentes

Motivación: Observamos que muchos almacenes aún llevan el control de inventario de manera manual, lo que puede generar errores, pérdidas de productos y falta de control en las entradas y salidas. Queríamos automatizar este proceso con un software accesible y funcional.

Definicion del problema: Falta de control en el inventario de almacenes de bebidas. Errores humanos en el registro de entradas y salidas de productos. Dificultad para identificar cuándo el stock está bajo o completo. No contar con un historial estructurado de movimientos.

Descripción del Proyecto

Tecnologias usadas:

IDE: Visual Studio Code

Lenguaje: Python

Bibliotecas: os, csv, datetime, qrcode y py2flowchart

Almacenamiento: CSV y Archivos de texto (txt)

Interfaz: Consola

Arquitectura del sistema: Diagramas de flujo

Funcionalidades principales:

  1. Creacion de codigos QR para acceso rapido a informacion del proyecto
  2. Creacion de diagramas de flujo
  3. CRUD en inventario (Create, Read, Update, Delete)
  4. Calculo de stock completo
  5. Alertas de stock bajo o completo
  6. Mostrar informacion general

Demostración

Desafíos y Soluciones

Problemas encontrados:

  1. Manejo de archivos para guardar registros sin corrupción de datos
  2. Optimización del menú para facilitar la navegación del usuario
  3. Implementación de funciones de seguridad para evitar datos inválidos
  4. Creacion del nombre de los archivos txt con la fecha y numero contador

Soluciones implementadas:

  1. Uso de archivos .txt y .csv con estructura clara y fácil de leer
  2. Validaciones en todas las entradas del usuario para evitar errores
  3. Registros automáticos en archivos de historial para seguimiento preciso
  4. Uso de la biblioteca datetime para obtener la fecha y hora actual ademas de crear una funcion para crear un contador y agregarlo al nombre del archivo

Futuro del Proyecto

Mejoras futuras:

  1. Implementación de una interfaz gráfica para mayor accesibilidad
  2. Base de datos en lugar de archivos de texto para mayor escalabilidad
  3. Integración con lectores de código de barras para agilizar el registro de productos

Potencial de escalabilidad:

  1. Podría expandirse para gestionar no solo bebidas, sino cualquier tipo de inventario
  2. Se podría convertir en un software con múltiples usuarios y acceso remoto

Conclusión

Aprendimos a trabajar con archivos en Python para almacenar datos de manera eficiente, segura y estructurada. Mejoramos nuestras habilidades en validaciones de entrada y manejo de errores. Nos dimos cuenta de la importancia de mantener un código limpio y estructurado para facilitar futuras mejoras, además del uso de librerias para añadir funcionalidades al proyecto.
